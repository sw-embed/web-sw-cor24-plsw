# UI/UX Design: web-sw-cor24-plsw

## Layout Overview

The UI follows the cor24-rs wizard/notebook pattern: a 3-column grid with
progressive disclosure. The user advances through compilation stages via a
wizard sidebar, with each stage's output appearing as a notebook cell that
scrolls into view.

Two tabs provide different views of the same compilation:
- **Editor tab**: wizard-driven workflow from source to execution
- **Pipeline tab**: 7-stage compiler internals visualization

## PL/SW Syntax Highlighting Scheme

### Keywords (--mauve / purple)
```
DCL DECLARE PROC PROCEDURE RETURNS DO END IF THEN ELSE WHILE
CALL GOTO RETURN STATIC AUTOMATIC EXTERNAL REGISTER BASED
BIT BYTE WORD INT CHAR PTR FIXED
```

### Macro Invocations (--peach / orange)
```
?GETMAIN  ?ESTAE  ?FREEMAIN  ?LINK  ?ENTRY  ?EXIT
```
Regex: `\?[A-Z][A-Z0-9_]*`

### Macro Definitions (--pink)
```
MACRODEF  MACROEND  GEN  ENDGEN
```

### Inline Assembly (--teal)
```
ASM DO; ... END;
```
The entire block is tinted with a subtle --teal background.

### GEN Blocks (--sky background tint)
Assembly code generated by macros, shown with a faint --sky background
to distinguish from handwritten assembly.

### String Literals (--green)
Single-quoted strings: `'Hello'`

### Numeric Literals (--peach)
Decimal, hex (0x...), binary (0b...)

### Comments (--overlay1 / dim)
```
/* block comments */
// line comments (if supported)
```

### Operators and Punctuation (--subtext1)
`;  (  )  ,  =  +  -  *  /  &  |  ^`

### Level Numbers (--yellow / bold)
```
DCL 1 RECORD,
      3 FIELD1 INT(24),
      3 FIELD2 CHAR(8);
```

## Macro Expansion Toggle Design

In the preprocessed output cell, each macro invocation is rendered as a
collapsible block:

```
  ?GETMAIN(LENGTH(256), SUBPOOL(0), ADDRESS(ptr));
  [v] Expansion (3 lines)                           <-- click to toggle
  |   LDA  r0, =256        ; LENGTH parameter
  |   LDA  r1, =0          ; SUBPOOL parameter
  |   JSR  r1, _svc_getmain
```

- Collapsed by default (shows invocation line only)
- Toggle arrow [>] / [v] on the expansion header
- Expansion lines indented with a left border in --peach
- GEN block output uses --sky left border
- Count of expanded lines shown in header

## Pipeline Stage Visualization (Tab 2)

The Pipeline tab shows all 7 compiler stages as vertical panels:

```
+--------------------------------------------------+
| Pipeline: hello.plsw                              |
+--------------------------------------------------+
|                                                    |
| [v] Stage 1: Macro Preprocessing                  |
| +------------------------------------------------+|
| | Input: .plsw + .msw files                      ||
| | Output: Expanded .plsw source                  ||
| | ?GETMAIN(...) -> [expanded 3 lines]            ||
| +------------------------------------------------+|
|                                                    |
| [v] Stage 2: Lexical Analysis                     |
| +------------------------------------------------+|
| | Token stream:                                   ||
| | DCL(kw) COUNT(ident) INT(kw) ((punc) 24(num)   ||
| +------------------------------------------------+|
|                                                    |
| [v] Stage 3: Parsing                              |
| +------------------------------------------------+|
| | AST tree (collapsible nodes):                   ||
| | Program                                         ||
| |   Declaration: COUNT                            ||
| |     Type: INT(24)                               ||
| |   Procedure: MAIN                               ||
| |     ...                                         ||
| +------------------------------------------------+|
|                                                    |
| [>] Stage 4: Semantic Analysis                     |
| [>] Stage 5: IR Lowering                           |
| [>] Stage 6: Register Allocation                   |
|                                                    |
| [v] Stage 7: Assembly Emission                     |
| +------------------------------------------------+|
| | _main:                                          ||
| |     LDA r0, =0                                  ||
| |     STA r0, [fp-3]   ; COUNT                    ||
| |     ...                                         ||
| +------------------------------------------------+|
+--------------------------------------------------+
```

Each stage panel:
- Collapsible with [>] / [v] toggle
- Shows stage name and brief description
- Input/output sections
- Stages 2-6 initially show "Awaiting compiler diagnostic support"
  and are populated as the compiler adds `-v` verbose output

## Debugger Panel Design

The debugger appears in the final notebook cell after assembly:

```
+--------------------------------------------------+
| Debugger: hello.plsw                              |
+--------------------------------------------------+
| [Step x1 v] [Run] [Stop] [Reset]  Speed [====--] |
| Status: RUNNING  Instructions: 12,345             |
+--------------------------------------------------+
| Assembly Listing      | Registers                 |
| 0000: _start:         | r0: 000042  *             |
| 0000:   LDA r0, =0    | r1: 000000                |
| 0003:   STA r0, [fp-3] | r2: FEE400               |
| 0006: > JSR r1, _main  | fp: FEE3F0               |
| 0009:   HLT            | sp: FEE3E0               |
| ...                    | z:  000000                |
|                        | iv: 000000                |
| (auto-scrolls to PC)  | ir: 000000                |
|                        | PC: 000006                |
|                        | C:  0                     |
|                        |--------------------------+|
|                        | UART TX:                  |
|                        | Hello, world!             |
|                        | UART RX: [___________]   |
|                        |--------------------------+|
|                        | SRAM (0x000-0xFFF)        |
|                        | 0000: 4C 44 41 20 ...     |
|                        | Stack (0xFEE000-FEE400)   |
|                        | FEE3F0: 00 00 09 ...      |
+--------------------------------------------------+
```

### Register Heatmap
- **Hot** (changed last step): --green background
- **Warm** (changed 2 steps ago): --yellow background, fading
- **Cold** (unchanged): no highlight

### Listing Auto-Scroll
- Current PC line highlighted with --yellow at 15% opacity
- `>` marker in gutter for current instruction
- Container scrolls to keep PC visible (1/3 from top)

## Keyboard Shortcuts

|-----|--------|
| Ctrl+Enter | Advance wizard (compile next stage) |
| F5 | Run program |
| Shift+F5 | Stop execution |
| F10 | Step (single instruction) |
| Shift+F10 | Step x10 |
| Ctrl+Shift+F10 | Step x100 |
| F6 | Reset emulator |
| Ctrl+1 | Switch to Editor tab |
| Ctrl+2 | Switch to Pipeline tab |
| Ctrl+O | Upload .plsw file |

## Responsive Behavior

- **Min width**: 1024px (3-column layout)
- **Below 1024px**: sidebar collapses to icons, wizard steps become a
  horizontal bar above notebook cells
- **Notebook cell padding-bottom**: `calc(100vh - 150px)` to allow any
  cell to scroll to the top of the viewport (same pattern as cor24-rs)
- **Debugger listing**: fixed height with internal scroll, register panel
  scrolls independently
