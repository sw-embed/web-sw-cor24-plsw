## Step 4: Read the base project research

Read ~/github/sw-embed/sw-cor24-plsw/docs/research.txt thoroughly. This is the
comprehensive design document for the PL/SW language. Key aspects:
- PL/S-inspired systems programming language for COR24
- Freestanding: no runtime requirement, direct OS service integration
- PL/I-like syntax with DCL/DECLARE and level-based declarations
- Transpiler approach: compiles to .s assembler (not .o object files)
- AST-aware macro preprocessor (m4-like, not CPP-style)
- Three levels of inline assembler escape hatch (asm! macro style)
- 7-layer compiler pipeline
- File types: .plsw source, .msw macro files
- COR24 ISA constraints: no floats, chars and ints only
- GEN blocks for register-level code generation
- Macro invocation syntax: ?GETMAIN(...), ?ESTAE(...), etc.
- Recovery/environment macros for exception-like handling
- Mixed listing output showing expanded source

Also read ~/github/sw-embed/sw-cor24-plsw/Cargo.toml and any other docs/ files
to understand the current state of the base PL/SW project.
